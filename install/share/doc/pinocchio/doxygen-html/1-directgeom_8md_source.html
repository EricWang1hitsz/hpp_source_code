3 ## Objective
4 
5 In this first series of exercises, we are going to start using the
6 library Pinocchio. We will load in the simulator the model of a simple
7 manipulator arm, the Universal Robot 5, or UR5. This model will be used
8 for a positioning task using simple methods. This set of exercices
9 emphasizes the first part of the class, about direct geometry.
10 
11 The software Pinocchio is a C++ library provided with a python wrapping
12 that allows us to control it with a python terminal. Let's see how it
13 works.

48 ### Pinocchio
49 
50 #### Basic mathematical objects:
51 
52 In the following, we will use numpy `Matrix` class to represent matrices
53 and vectors. In numpy, vectors simply are matrices with one column. See
54 the following example.
55 
56 ```py
57 import numpy as np
58 A = np.matrix([[1, 2, 3, 4], [5, 6, 7, 8]]) # Define a 2x4 matrix
59 b = np.zeros([4, 1]) # Define a 4 vector (ie a 4x1 matrix) initialized with 0
60 c = A * b # Obtain c by multiplying A by b.
61 ```
62 
63 A bunch of useful functions are packaged in the utils of pinocchio.
64 
65 ```py
66 from pinocchio.utils import *
67 eye(6) # Return a 6x6 identity matrix
68 zero(6) # Return a zero 6x1 vector
69 zero([6, 4]) # Return az zero 6x4
70 matrix rand(6) # Random 6x1 vector
71 isapprox(zero(6), rand(6)) # Test epsilon equality
72 mprint(rand([6, 6])) # Matlab-style print
73 skew(rand(3)) # Skew "cross-product" 3x3 matrix from a 3x1 vector
74 cross(rand(3), rand(3)) # Cross product of R^3
75 rotate('x', 0.4) # Build a rotation matrix of 0.4rad around X.
76 ```
77 
78 Specific classes are defined to represent objects of \f$SE(3)\f$, \f$se(3)\f$ and
79 \f$se(3)^*\f$. Rigid displacements, elements of \f$SE(3)\f$, are represented by
80 the class `SE3`.
81 
82 ```py
83 import pinocchio as se3
84 R = eye(3); p = zero(3)
85 M0 = se3.SE3(R, p)
86 M = se3.SE3.Random()
87 M.translation = p; M.rotation = R
88 ```
89 
90 Spatial velocities, elements of \f$se(3) = M^6\f$, are represented by the
91 class `Motion`.
92 
93 ```py
94 v = zero(3); w = zero(3)
95 nu0 = se3.Motion(v, w)
96 nu = se3.Motion.Random()
97 nu.linear = v; nu.angular = w
98 ```
99 
100 Spatial forces, elements of \f$se(3)^* = F^6\f$, are represented by the
101 class `Force`.
102 
103 ```py
104 f = zero(3); tau = zero(3)
105 phi0 = se3.Force(f, tau)
106 phi = se3.Force.Random()
107 phi.linear = f; phi.angular = tau
108 ```

110 ## 1.1) Creating and displaying the robot
111 
112 ### Robot kinematic tree
113 
114 The kinematic tree is represented by two C++ objects called Model (which
115 contains the model constants: lengths, masses, names, etc) and Data
116 (which contains the working memory used by the model algorithms). Both
117 C++ objects are contained in a unique Python class. The first class is
118 called RobotWrapper and is generic.
119 
120 For the next steps, we are going to work with the RobotWrapper.
121 
122 Import the class `RobotWrapper` and create an instance of this class in
123 the python terminal. At initialization, RobotWrapper will read the model
124 description in the URDF file given as argument. In the following, we
125 will use the model of the UR5 robot, available in the directory "models"
126 of pinocchio (available in the homedir of the VBox).
127 
128 ```py
129 from pinocchio.robot_wrapper import RobotWrapper
130 from os.path import join
131 
132 PKG = '/opt/openrobots/share'
133 URDF = join(PKG, 'ur5_description/urdf/ur5_gripper.urdf')
134 robot = RobotWrapper(URDF, [PKG])
135 ```
136 
137 The code of the RobotWrapper class is in
138 `/opt/openrobots/lib/python2.7/site-packages/pinocchio/robot_wrapper.py`.
139 Do not hesitate to have a look at it and to take inspiration from the
140 implementation of the class functions.
141 
142 UR5 is a fixed robot with one 6-DOF arms developed by the Danish company
143 Universal Robot. All its 6 joints are revolute joints. Its configuration
144 is in R^6 and is not subject to any constraint. The model of UR5 is
145 described in a URDF file, with the visuals of the bodies of the robot
146 being described as meshed (i.e. polygon soups) using the Collada format
147 ".dae". Both the URDF and the DAE files are available in the package
148 `robotpkg-ur5-description`
149 
150 ### Exploring the model
151 
152 The robot model is available in `robot.model`. It contains the names of
153 all the robot joint `names`, the kinematic tree `parents` (i.e. the
154 graph of parents, 0 being the root and having no parents), the position
155 of the current joint in the parent coordinate frame `jointPosition`,
156 the mass, inertia and center-of-gravity position of all the bodies
157 (condensed in a spatial inertia 6x6 matrix) `inertias` and the gravity
158 of the associated world `gravity`. All these functions are documented
159 and are available in the correponding class dictionnary.
160 
161 ```py
162 for name, function in robot.model.__class__.__dict__.items():
163  print(" \*\*\*\* ", name, ": ", function.__doc__)
164 ```
165 
166 Similarly, the robot data are available in `robot.data`. All the variables
167 allocated by the classical rigid-body dynamics algorithms are stored in
168 `robot.data` and are available through the python wrapping. Similarly to
169 the model object, the function are documented and are available from the
170 class dictionnary. The most useful in the following will be the
171 placement of the frame associated which each joint output stored in
172 `robot.data.oMi`.
173 
174 For example, the robot end effector corresponds to the output of the
175 last joint, called `wrist_1_joint`. The ID of the joint in the joint
176 list can be recovered from its name, and then used to access its
177 placement:
178 
179 ```py
180 # Get index of end effector
181 
182 idx = robot.index('wrist_3_joint')
183 
184 # Compute and get the placement of joint number idx
185 
186 placement = robot.position(q, idx)
187 # Be carreful, Python always returns references to values.
188 # You can often .copy() the object to avoid side effects
189 # Only get the placement
190 placement = robot.data.oMi[idx].copy()
191 ```
192 
193 Finally, some recurring datas (used in Model and Data) have been wrapped
194 to functions in some python shortcuts, also available in RomeoWrapper:
195 
196 - The size of the robot configuration is given by `nq`.
197 - The dimension of its tangent space (velocity) is `nv`.
198 - The index of a joint in the tree can be accessed from its name by index (see above).
199 - The classical algorithms are also binded: com, Jcom, mass, biais, joint gravity, position and velocity of each joint.
200 
201 ```py
202 q = zero(robot.nq)
203 v = rand(robot.nv)
204 robot.com(q) # Compute the robot center of mass.
205 robot.position(q, 3) # Compute the placement of joint 3
206 ```

208 ### Display the robot
209 
210 To display the robot, we need an external program called *Gepetto
211 Viewer*. If you completed the installation in the previous page, you can
212 launch this program, open a new terminal in an empty workspace.
213 
214 ```
215 student@student-virtualbox:~$ gepetto-gui
216 ```
217 
218 This will start a server waiting for instructions. We will now create a client that will
219 ask the server to perform some requests (such as creating a window or
220 displaying our robot)
221 
222 In a python terminal you can now load the visual model of the robot in
223 the viewer:
224 
225 ```py
226 robot.initDisplay(loadModel=True)
227 ```
228 
229 This will flush the robot model inside the GUI. The argument
230 `loadModel=True` is mandatory when you start or restart the GUI. In
231 later call to your scripts, you can set the argument to `False`. A side
232 effector of `=True` is that it will move the viewpoint inside the GUI to
233 a reference zero position.
234 
235 ### More details about loading the model (optionnal)
236 
237 You can access the visual object composing the robot model by
238 `robot.visual_model.geometryObject`.
239 
240 ```py
241 visualObj = robot.visual_model.geometryObjects[4] # 3D object representing the robot forarm
242 visualName = visualObj.name # Name associated to this object
243 visualRef = robot.getViewerNodeNames(visualObj) # Viewer reference (string) representing this object
244 ```
245 
246 Moving one object
247 
248 ```py
249 q1 = (1, 1, 1, 1, 0, 0, 0) # x, y, z, quaternion
250 robot.viewer.gui.applyConfiguration(visualRef, q1)
251 robot.viewer.gui.refresh() # Refresh the window.
252 ```
253 
254 Additional objects can be created, like a sphere as follows.
255 
256 ```py
257 rgbt = [1.0, 0.2, 0.2, 1.0] # red, green, blue, transparency
258 robot.viewer.gui.addSphere("world/sphere", .1, rgbt) # .1 is the radius
259 ```
260 
261 The exhaustive list of the object that can be created is available in
262 the IDL of the GUI:
263 `/opt/openrobots/share/idl/gepetto/corbaserver/graphical-interface.idl`

265 ## 1.2) Simple pick and place
266 
267 *Objectives:* Display the robot at a given configuration or along a
268 given trajectory
269 
270 ### Pick:
271 
272 Say we have a target at position `[.5, .1, .2]` and we would like the
273 robot to grasp it.
274 
275 ```py
276 robot.viewer.gui.applyConfiguration("world/sphere", (.5, .1, .2, 1.,0.,0.,0. ))
277 robot.viewer.gui.refresh() # Refresh the window.
278 ```
279 
280 First display a small sphere at this position to visualize it.
281 
282 Then decide by any mean you want a configuration of the robot so that
283 the end effector is touching the sphere.
284 
285 At the reference position you built, the end effector placement can be
286 obtained by `robot.position(q, 6)`. Only the translation part of the
287 placement has been selected. The rotation is free.
288 
289 *Optional* Say now that the object is a rectangle and not a sphere.
290 Pick the object at a reference position with the rotation that is
291 imposed, so that the end effector is aligned with one of the faces of
292 the rectangle.
293 
294 ### Place:
295 
296 Choose any trajectory you want in the configuration space, starting from
297 the reference position built in the previous exercice (it can be
298 sinus-cosinus waves, polynomials, splines, straight lines).
299 
300 Make a for loop to display the robot at sampling positions along this
301 trajectory. The function sleep in module time (from time import sleep)
302 can be used to slow down the loop.
303 
304 At each instant of your loop, recompute the position of the ball and
305 display it so that it always "sticks" to the robot end effector.
